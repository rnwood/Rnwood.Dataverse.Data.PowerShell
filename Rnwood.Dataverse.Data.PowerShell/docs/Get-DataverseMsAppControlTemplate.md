# Get-DataverseMsAppControlTemplate

## SYNOPSIS
Lists Canvas control templates from embedded all-controls metadata, optionally augmented by a .msapp file.

## SYNTAX

### Embedded (Default)
```
Get-DataverseMsAppControlTemplate [-TemplateName <String>] [-ProgressAction <ActionPreference>]
 [<CommonParameters>]
```

### FromPath
```
Get-DataverseMsAppControlTemplate [-MsAppPath] <String> [-TemplateName <String>]
 [-ProgressAction <ActionPreference>] [<CommonParameters>]
```

### FromObject
```
Get-DataverseMsAppControlTemplate [-CanvasApp] <PSObject> [-TemplateName <String>]
 [-ProgressAction <ActionPreference>] [<CommonParameters>]
```

## DESCRIPTION
Returns control template metadata used by Canvas apps, including template name, version, template ID, YAML control name,
variant requirements, available variants, and app preview flag requirements.

By default, templates are read from the embedded `All controls.msapp` catalog.
If `-MsAppPath` or `-CanvasApp` is provided, those templates are merged with embedded templates.
When the same template exists in multiple sources, only the latest version is returned.

## EXAMPLES

### Example 1: List all known templates from embedded catalog
```powershell
Get-DataverseMsAppControlTemplate
```

### Example 2: Include templates from a local .msapp file
```powershell
Get-DataverseMsAppControlTemplate -MsAppPath "C:\apps\myapp.msapp"
```

### Example 3: Filter templates by name
```powershell
Get-DataverseMsAppControlTemplate -TemplateName "*button*"
```

### Example 4: Use a Canvas app object from pipeline
```powershell
$app = Get-DataverseCanvasApp -Name "new_myapp" -IncludeDocument
$app | Get-DataverseMsAppControlTemplate
```
